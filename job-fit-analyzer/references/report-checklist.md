# Report Checklist

Use this checklist **before** presenting the final HTML report. Fix any failed items and re-check until all pass.

## Required Checks

- [ ] JD Quality Assessment (including the 7 checks and final verdict) appears **before** the match score.
- [ ] All seven JD quality checks have **specific, evidence-based findings** (no generic statements).
- [ ] Problem Diagnosis section includes clear business problem mapping and candidate attack points.
- [ ] Overall score exactly matches the weighted sum of the five dimension scores (after penalties).
- [ ] Report uses **exactly** the five defined dimensions (no more, no less).
- [ ] Every dimension score includes: formula, sub-scores, key evidence, and clear rationale.
- [ ] Experience dimension explicitly shows years, seniority, relevance, and any penalties applied.
- [ ] ATS Keyword Coverage distinguishes between "demonstrated in experience" vs "listed only", and highlights priority keywords.
- [ ] Strengths and Weaknesses are **specific** to this JD + resume (not generic).
- [ ] Recommendations contain **exactly three** categories: Resume Tailoring, Skill Gap Mitigation, and Application Strategy.
- [ ] No interview preparation content appears in the recommendations (this belongs to interview-coach).
- [ ] All placeholders from `assets/report-template.html` have been replaced with real content.
- [ ] Language of the report matches the dominant language of the JD (Chinese or English).
- [ ] Report is saved as `job-fit-report.html` (or user-specified name).

## Fix Loop

1. Record each failed item and the affected section.
2. Correct the content, calculations, or structure.
3. Recalculate scores if any evidence or dimension scores changed.
4. Re-run the full checklist until **all items pass**.

**Final Step**: Only after passing 100% of the checklist, present the report to the user.
